# &str vs str

아래와 같이 러스트에서 정의한 문자열은 str이 아니라 &str을 타입으로 가집니다.
str 타입을 바로 사용할 수 없는 이유를 고르시오:

```rs
fn main() {
    let message: str = "저 넓은 세상에서 큰 꿈을 펼쳐라"; // 컴파일 오류!
}
```

- [ ] str은 컴파일러만 사용할 수 있는 키워드이기 때문.
- [ ] 동적으로 할당한 문자열과의 일관성을 지키기 위해.
- [x] str은 크기를 알 수 없는 타입이기 때문.
- [ ] str은 가변 문자열이기 때문.
